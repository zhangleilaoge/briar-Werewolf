# 压力系统（Pressure System）

> **压力是记忆的产物。** 不是独立系统，在新记忆产生时计算一次。压力值满时触发状态判定（美德/崩溃）。

## 核心原则

- 压力是**记忆系统的衍生物**，不是独立系统
- 压力值**0~20**，只在**新记忆产生时计算一次**
- 压力满时触发**状态判定**（美德/崩溃），产生**状态**（State）
- 压力受**性格加权**影响
- 状态影响行为，但不是机制修改

## 压力值来源

新记忆产生时，根据以下事件类型计算压力增量：

| 事件类型 | 条件 | 基础压力 | 说明 |
|----------|------|----------|------|
| `hear_claim` | `actorId` = me，`claimedRole` ≠ 真实角色 | +5 | 说谎/扮演其他角色 |
| `hear_accuse` | `targetId` = me | +3 | 被指控 |
| `vote` | `targetId` = me | +2 | 被投票 |
| `hear_defend` | `actorId` = me，`targetId` = 队友 | +2 | 保护队友（狼人） |
| `check_result` | 被查验为狼人（如果该信息被公开） | +4 | 被查杀（知道的情况下） |

> **压力计算只发生在和「我」相关的记忆上**。别人指控别人、别人说别人的事，不会增加我的压力。

## 压力计算流程

```
新记忆产生
  ↓
判断记忆是否和「我」相关（actorId = me 或 targetId = me）
  ↓
获取基础压力值（如上表）
  ↓
乘以性格加权系数
  ↓
加到当前压力值上
  ↓
检查是否≥20（满）
```

## 性格加权系数

不同性格对同一事件的压力反应不同：

| 性格 | 被指控 | 被投票 | 说谎 | 保护队友 | 特点 |
|------|--------|--------|------|----------|------|
| **好斗型** | 0.5 | 0.7 | 1.0 | 1.0 | 被指控不怕 |
| **谨慎型** | 1.2 | 1.3 | 1.5 | 1.0 | 说谎非常紧张 |
| **操控型** | 0.8 | 0.8 | 0.5 | 1.0 | 说谎很淡定 |
| **忠诚型** | 1.0 | 1.0 | 1.0 | 0.5 | 保护队友理所当然 |
| **多疑型** | 0.7 | 0.8 | 1.0 | 1.2 | 被指控习惯了，但为别人辩护紧张 |

## 状态判定（State Resolution）

当压力值达到20时，触发状态判定。状态是**行为修饰**，不是机制修改。

### 第一次满（压力≥20）

1. 清空压力值 → 0
2. 判定：
   - **美德（Virtue）**：25%概率 → 获得**正面状态**
   - **崩溃（Breakdown）**：75%概率 → 获得**负面状态**
3. 状态生效（详见 [STATE.md](STATE.md)）

### 第二次满（压力再次≥20）

1. 清空压力值 → 0
2. **失去角色推理能力**：`InferenceEngine.inferAll` 和 `inferPlayer` 返回默认概率
3. **失去局势推理能力**：`InferenceEngine.inferSelfCrisis` 和 `inferFieldCrisis` 返回默认值
4. **暴露风险**：如果是狼人，有概率（30%）直接说出自己的真实身份

### 超过两次满

每次满都会重复第二次满的效果（清空压力 + 失去推理 + 可能暴露）。

## 压力与状态的关系

```typescript
interface PressureState {
  current: number;     // 当前压力值，0~20
  max: number;        // 压力上限，默认20
  burstCount: number; // 已满次数
  states: State[];    // 已获得的状态列表
}
```

- 正常状态（burstCount = 0）：推理正常
- 第一次满后（burstCount = 1）：有美德/崩溃状态，但推理正常
- 第二次满后（burstCount ≥ 2）：**失去推理能力**

`InferenceEngine` 在推理前检查：`if (player.burstCount >= 2) return 默认概率`

## 状态 vs 特质

| | 状态（State） | 特质（Trait） |
|---|---|---|
| 来源 | 压力满（美德/崩溃） | 职业默认（如孤狼）或机制触发 |
| 作用 | 影响行为/属性（如攻击欲、推理准确度） | 影响机制（如击杀协调规则） |
| 可叠加 | 可以（多个状态） | 可以（多个特质） |
| 消失 | 否（永久） | 否（永久） |
| 例子 | 偏执（suspect×1.5）、坚韧（压力上限+5） | 孤狼（独立击杀协调） |

## 设计原则

1. **压力不是惩罚，是角色深度**：压力让AI有弱点，更像人
2. **性格影响压力**：不同性格对同一事件反应不同
3. **满有代价也有机会**：美德可以变强，崩溃会变弱
4. **两次满是硬限制**：失去推理能力意味着AI只能靠记忆硬信息做决策
5. **状态是行为修饰，不是机制修改**：状态影响行动权重，但不改变游戏规则
