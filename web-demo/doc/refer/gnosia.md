# 古诺西亚（GNOSIA）参考

本文档汇总科幻狼人杀游戏 **GNOSIA（古诺西亚 / 古诺希亚）** 的白天流程、角色身份与可使用的讨论指令，供本项目设计参考。

> GNOSIA 由 Petit Depotto 开发，融合了日式人狼、视觉小说与 RPG 数值成长要素。每场约 15 分钟，玩家在多周目轮回中逐步解锁角色背景与技能。

---

## 核心循环

游戏以“轮回”为单位，每个轮回包含多个昼夜循环：

1. **白天（会议 / 讨论阶段）**
   - 所有存活角色参与讨论，质疑、袒护、发动技能。
   - 讨论结束后进行投票，票数最高者被送入“冷冻睡眠”（出局）。
   - 玩家一旦被投票休眠，该轮回立即结束，进入下一轮。
2. **夜晚（行动阶段）**
   - 古诺西亚选择一名乘客抹杀。
   - 工程师、医生、守护天使等角色按顺序发动夜间能力。
   - 第二天公布死亡/消失信息。

> 与传统狼人杀不同：GNOSIA **没有“遗言”环节**，也没有天亮后让死亡玩家描述昨夜行动的机制；推理主要依赖白天讨论、技能结果和角色关系。

## 角色身份（共 8 种）

| 身份 | 对应传统狼人杀 | 阵营 | 技能 |
|------|---------------|------|------|
| 乘客 | 平民 | 好人 | 无特殊技能，白天投票，夜间可能被古诺西亚抹杀 |
| 古诺西亚 | 狼人 | 狼人 | 每晚抹杀一名乘客；不能杀古诺西亚、不能杀 BUG |
| 工程师 | 预言家 | 好人 | 每晚查验一名玩家是否为古诺西亚；次日可通报结果 |
| 医生 | 灵媒师 / 守墓人 | 好人 | 每晚查验上一白天被投票休眠玩家的阵营 |
| 守护天使 | 守卫 / 狩人 | 好人 | 每晚保护一名玩家免受古诺西亚袭击，通常不能保护自己 |
| 留舰者 | 共有者 | 好人 | 场上有两人；一人表明身份时另一人同时公开；不能被冒充 |
| AC主义者 | 隐狼 / 狂人 | 狼人侧 | 被查验显示为人类；可冒充工程师或医生；与古诺西亚互不知身份 |
| BUG | 第三方 | 第三方 | 可冒充工程师或医生；胜利条件为不被工程师查验到；查验到则消失 |

### 胜利条件

| 阵营 | 胜利条件 |
|------|---------|
| 乘客（好人） | 投票休眠所有古诺西亚，且 BUG 不在场 |
| 古诺西亚（狼人） | 古诺西亚数量 ≥ 乘客数量，且 BUG 不在场 |
| AC主义者 | 古诺西亚胜利时自己存活 |
| BUG | 不被工程师查验到，存活到最后 |

## 白天讨论流程

### 1. 早晨事件结算

- 公布昨夜死亡/消失的玩家（若有）。
- 工程师、医生可选择在此时公布昨夜查验结果。
- 若有玩家被投票休眠，医生可查验其阵营。

### 2. 自由讨论

- 玩家依次发言，可质疑、袒护、反驳他人。
- 角色好感度会影响他人是否相信或支持你。
- 玩家可使用“指令”影响讨论走向。

### 3. 投票

- 讨论结束后，所有存活玩家投票。
- 票数最高者被送入冷冻睡眠（出局）。
- 平票时通常进入下一轮讨论或按特殊规则处理。

## 白天可用指令/技能

GNOSIA 的白天指令具有 RPG 属性要求，部分需要满足条件才能解锁。

| 指令/技能 | 属性要求 | 效果 |
|-----------|---------|------|
| 质疑 / 怀疑 | — | 攻击某玩家可疑，引导他人怀疑目标 |
| 袒护 | — | 保护某玩家，降低其被怀疑程度 |
| 感谢 | — | 被袒护后使用，提升好感度 |
| 反驳 | 逻辑性 | 打破袒护氛围，继续攻击目标；受逻辑性和演技影响 |
| 一同反驳 | — | 加入他人反驳，轻微追加攻击 |
| 亮明身份 | — | 公开自己的身份（如工程师），骗子可谎称身份；每天限一次 |
| 别上当了 | 直觉 30 | 被攻击后使用，将注意力转移到攻击者身上，持续到次日报告结束 |
| 投票给此人 | 逻辑性 10 | 提议大家投票给工程师指出的古诺西亚，或已确定的敌人；效果仅当天 |
| 别投票给此人 | 逻辑性 25 | 阻止大家投票给某人；每天限一次 |
| 此人定是人类 | 逻辑性 20 | 强烈保证某人为人类，使其基本不再被怀疑；搞错会遭反感 |
| 此人定是敌人 | 逻辑性 20 | 强烈指认某人为敌人，被指出者基本出局；必杀但风险高 |
| 排除全员 | 逻辑性 30 | 当某身份有两人以上自称时，提议全部送去冷冻；每轮回限一次 |
| 闲聊一下 | 隐蔽性 10 | 转移话题，降低自己被古诺西亚袭击概率，提升闲聊者好感；每天限一次 |
| 配合闲聊 | — | 加入闲聊，进一步缓和气氛 |
| 阻止闲聊 | — | 阻止当前闲聊 |
| 我们合作吧 | 亲和力 15 | 与一名玩家建立合作关系，成为坚定同伴；每天限一次 |
| 发誓自己是人类 | 直觉 20 | 让所有人发誓自己是人类，非人类者必须撒谎；每轮回限一次 |
| 跪地求饶 | 隐蔽性 35 | 被投票时求饶，可能避免被冷冻；每轮回限一次 |

## 六维属性系统

GNOSIA 中角色拥有六项属性，随轮回积累经验成长：

| 属性 | 作用 |
|------|------|
| 亲和力 | 说服、合作、建立好感 |
| 逻辑性 | 推理、质疑、排除全员等指令 |
| 直觉 | 识破谎言、别上当了、发誓自己是人类 |
| 演技 | 撒谎、伪装、跪地求饶 |
| 领导力 | 影响赞同/跟票人数、主导讨论 |
| 隐蔽性 | 降低被袭击概率、闲聊、跪地求饶 |

> 属性成长是 GNOSIA 的核心 RPG 要素之一，与本项目的“六项属性 + 骰子检定”设计有较高参考价值。

## 与本地机制的对应启发

| GNOSIA 机制 | 本项目对应 |
|-------------|-----------|
| 白天讨论 + 投票 | 白天阶段：公开讨论、私下交流、投票放逐 |
| 六维属性 | 六项角色属性：亲和、逻辑、洞察、领导、诡诈、隐蔽 |
| 指令解锁与属性要求 | 白天行动需要属性达到一定阈值或进行检定 |
| 合作关系 | 结盟 / 友好值 / 信任值系统 |
| 好感度影响信任 | 关系值系统 |
| 古诺西亚夜间抹杀 | 夜晚杀戮阶段 |
| 工程师查验 | 信息获取类能力 |
| 医生查验被放逐者 | 死亡信息/阵营揭示类能力 |
| BUG 第三方胜利 | 独立胜利条件的第三方角色 |
