# 游戏机制主文档

## 核心定位

- 本游戏不是单纯的狼人杀游戏。
- 也不是完全基于约束满足问题（Constraint Satisfaction Problem, CSP）的设计。
- 游戏中存在大量变量和数值。
- 许多行动的效果由数值与骰子共同决定。

## 文档结构

- `main.md`：机制总览与核心设计原则。
- `flow.md`：白天与夜晚的对局流程。
- `numeric.md`：属性、关系、检定等全部数值细节。

## 角色属性（概述）

每个角色拥有五项核心属性：亲和、逻辑、领导、诡诈、隐蔽。

**重要限制：所有属性只影响白天的投票与讨论阶段，不影响晚上的杀戮阶段。**

详见 [numeric.md](numeric.md) 中的「角色属性」。

## 角色关系（概述）

每对角色之间存在**有向关系**，n 名玩家共产生 `n × (n - 1)` 条关系。
每条关系包含两个数值：**信任值**与**友好值**。

关系只影响白天的互动与决策，不影响夜晚杀戮阶段。

详见 [numeric.md](numeric.md) 中的「角色关系」。

## 检定与骰子（概述）

白天行动使用「属性 + 骰子」的检定方式，部分行动会结合关系值修正难度。

详见 [numeric.md](numeric.md) 中的「检定规则」。

## 对局流程（概述）

对局由白天阶段与夜晚阶段交替组成：

- **白天阶段**：讨论、行动、投票放逐。
- **夜晚阶段**：角色能力、阵营行动与杀戮结算（独立于属性和关系）。

详见 [flow.md](flow.md)。
