# 数值设计

本文档记录游戏中所有数值细节，包括角色属性、角色关系、压力值与检定规则。

---

## 角色属性

### 范围

- 六项核心属性：亲和、逻辑、领导、诡诈、隐蔽、洞察。
- 数值通常在 **1–10** 之间，可随成长、状态或道具变化。
- **所有属性只影响白天阶段**，不影响夜晚杀戮。

### 属性作用

| 属性 | 英文 | 主要作用 |
|------|------|----------|
| 亲和 | Affinity | 说服、建立信任、降低他人压力、获取信息 |
| 逻辑 | Logic | 推理、识破谎言、构建证据链、保持理性投票 |
| 领导 | Leadership | 主导议程、影响跟票、组织公开行动 |
| 诡诈 | Deception | 撒谎、伪装、误导、陷害 |
| 隐蔽 | Stealth | 隐藏立场、降低被怀疑概率、隐藏压力状态 |
| 洞察 | Insight | 感知情绪动机、看穿伪装、观察破绽、读取压力 |

> 逻辑侧重从事实证据发现矛盾；洞察侧重从语气、表情和行为动机读取隐藏信息。

## 角色关系

### 规模与数值

- 每对角色之间存在**有向关系**，n 名玩家共产生 `n × (n - 1)` 条关系。
- 每条关系包含两个独立数值：

| 数值 | 英文 | 含义 | 范围 |
|------|------|------|------|
| 信任值 | Trust | 对对方可靠性与诚实的理性判断 | -10 ~ +10 |
| 友好值 | Friendliness | 对对方的情感倾向与好感 | -10 ~ +10 |

### 对白天阶段的影响

- **信任值**：影响相信言论、信息共享、跟票倾向、辩护意愿与说服难度。
- **友好值**：影响帮助意愿、保护倾向、结盟基础、情绪反应与信任恢复速度。

### 关系变化

白天事件改变角色之间的关系值：

| 事件 | 信任值变化 | 友好值变化 |
|------|------------|------------|
| 被证实说真话 / 共同正确推理 | +1 ~ +3 | +0 ~ +1 |
| 公布身份后被证实为真 | +1 ~ +3 | +0 ~ +1 |
| 公开信息被证实为真 | +1 ~ +2 | +0 ~ +1 |
| 公开信息被证实为假 | -2 ~ -4 | -1 ~ -2 |
| 被公开不利信息 | -1 ~ -2 | -1 ~ +0 |
| 被对方帮助 / 辩护 | +1 ~ +2 | +1 ~ +3 |
| 投票一致 | +0 ~ +1 | +0 ~ +1 |
| 被怀疑 | -1 ~ -2 | -1 ~ +0 |
| 被一同怀疑 | -1 ~ -2 | -1 ~ +0 |
| 反驳成功（目标获旁观者信任） | +1 ~ +2 | +0 ~ +1 |
| 反驳失败（目标失旁观者信任） | -2 ~ -4 | -1 ~ -2 |
| 被成功反驳（怀疑者失旁观者信任） | -1 ~ -2 | -1 ~ +0 |
| 被袒护 | +0 ~ +1 | +1 ~ +3 |
| 被一同袒护 | +0 ~ +1 | +1 ~ +2 |
| 被感谢 | +0 ~ +1 | +1 ~ +2 |
| 被号召投票 | -1 ~ -2 | -1 ~ +0 |
| 被阻止投票 | +0 ~ +1 | +0 ~ +1 |
| 被担保清白 | +2 ~ +4 | +1 ~ +3 |
| 担保清白正确 | +2 ~ +4 | +0 ~ +2 |
| 担保清白带错 | -4 ~ -6 | -2 ~ -4 |
| 被强烈指认 | -2 ~ -4 | -2 ~ -3 |
| 强烈指认正确 | +2 ~ +4 | +0 ~ +1 |
| 强烈指认错误 | -4 ~ -6 | -2 ~ -4 |
| 被提议全员排除 | -2 ~ -4 | -2 ~ -3 |
| 观察被目标发现 | -1 ~ -3 | -1 ~ -2 |
| 观察被其他玩家发现 | 0 | 0 |
| 共同怀疑 / 共同袒护（盟友间） | +0 ~ +1 | +0 ~ +1 |
| 被怀疑但未证实 | -1 ~ +0 | -1 ~ +0 |
| 被证实撒谎 / 误导 | -2 ~ -5 | -1 ~ -3 |
| 公布身份后被证实为假 | -2 ~ -5 | -1 ~ -3 |
| 背叛联盟 / 背后投票 | -3 ~ -6 | -2 ~ -5 |
| 被投票放逐（未成功） | -1 ~ -3 | -1 ~ -2 |

每天关系值变化设有上限，避免极端波动。

> 怀疑、袒护、公布身份等行动若后续被证实正确或错误，旁观者对行动者的关系变化可参照「被证实说真话」或「被证实撒谎」处理。

阵营对关系变化的修正详见 [alignment.md](alignment.md)。

## 压力值

### 定义与范围

- 压力值（Stress）是每名角色的独立心理状态数值，范围 **-10 ~ +10**。
- **0** 为基线状态；负值表示相对冷静/放松；正值表示紧张/焦虑。

| 区间 | 状态 |
|------|------|
| -10 ~ -7 | 极度冷静，甚至可能显得冷漠或迟钝 |
| -6 ~ -3 | 冷静，行为稳定，抗压能力强 |
| -2 ~ +2 | 正常基线，情绪波动在可控范围 |
| +3 ~ +5 | 轻微紧张，可能出现小破绽 |
| +6 ~ +8 | 明显焦虑，撒谎与伪装难度上升 |
| +9 ~ +10 | 高度紧张，极易暴露 |
| **+10** | **过载**：触发过载反应 |

### 影响

- **伪装与撒谎**：压力值越高（尤其为正值时），诡诈检定有效值越低（约压力值 × 0.5 的减值）。
- **隐藏立场**：压力值越高，隐蔽检定有效值越低。
- **观察破绽**：高压力角色更容易被观察读出异常。
- **投票行为**：压力过高时，投票更情绪化，理性判断下降。
- **过载反应**：压力值达到 **+10** 时触发过载，具体反应待设计。

> **TODO**：过载反应的具体效果与行为树尚未设计，需后续补充。

### 变化

| 事件 | 压力值变化 |
|------|------------|
| 被怀疑 | +1 ~ +2 |
| 被一同怀疑 | +1 ~ +2 |
| 被强烈指认 | +2 ~ +4 |
| 被号召投票 | +1 ~ +2 |
| 被担保清白 | -1 ~ -2 |
| 被阻止投票 | -1 ~ -2 |
| 被袒护 | -1 ~ -1 |
| 被一同袒护 | -1 ~ -1 |
| 被感谢 | -1 ~ +0 |
| 观察被发现 | +1 ~ +2 |
| 撒谎成功且未被发现 | +1 ~ +2 |
| 撒谎被识破 | +3 ~ +5 |
| 作为狼人参与夜晚击杀 | +1 ~ +2 |
| 投票放逐失败（目标未出局） | +1 ~ +2 |
| 每回合自然恢复 | 向 0 回归 1 ~ 2 |

压力值每天变化设有上限，每回合向基线值 **0** 回归：高于 0 时下降，低于 0 时上升，等于 0 时不变。下限 -10，上限 10。

### 与属性/阵营的互动

- **亲和**：自身恢复更快，降低他人压力的效果更强。
- **隐蔽**：更善于隐藏压力状态。
- **洞察**：更容易从他人压力中读出信息。
- **诡诈**：高压力下仍能保持一定伪装能力。
- **逻辑**：高压力下仍能保持相对理性投票。

阵营对压力值的修正详见 [alignment.md](alignment.md)。

## 检定规则

### 基本检定

```
检定结果 = 对应属性值 + d20
难度 = 目标对应属性值 或 固定难度值（如 15）
```

### 成功等级

| 结果 | 条件 | 效果 |
|------|------|------|
| 大失败 | 检定结果 ≤ 难度 - 10 | 行动失败，并可能触发负面后果 |
| 失败 | 检定结果 < 难度 | 行动未达成预期效果 |
| 成功 | 检定结果 ≥ 难度 | 行动达成预期效果 |
| 大成功 | 检定结果 ≥ 难度 + 10 | 行动获得额外收益或隐藏信息 |

### 对抗检定

| 行动 | 检定公式 |
|------|----------|
| 说服投票 | 己方亲和 + d20 vs 对方逻辑 + d20 |
| 识破谎言 | 己方逻辑 + d20 vs 对方诡诈 + d20 |
| 隐藏立场 | 己方隐蔽 + d20 vs 对方逻辑 + d20 |
| 观察 | 己方洞察 + d20 vs 对方隐蔽 + d20 |
| 反观察（是否被发现） | 己方隐蔽 + d20 vs 对方洞察 + d20 |

### 含关系修正的检定

```
说服难度 = 基础难度 + 对方逻辑 - 对方对自己的信任值
己方检定 = 己方亲和 + d20
```

检定成功可能进一步改变双方关系值（友好值小幅上升，信任值根据内容变化）。
