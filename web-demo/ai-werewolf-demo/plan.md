# 狼人杀 Demo 重构计划

## 现状分析

当前代码仅实现了最基础的：
- 简单的玩家角色分配（职业）
- 顺序错误的回合流程（白天→投票→夜晚，应为：夜晚→白天→投票）
- 极少的行动类型（speak, claim_check, berserker_kill, vote）
- 简化的信念系统（0~1范围，而非文档要求的 -10~+10）
- 简化的压力系统（0~1，而非 -10~+10）
- 关系系统极度简化
- 缺少属性系统、阵营系统、检定系统（d20）
- 缺少追加行动（一同怀疑、反驳、一同袒护）
- 缺少连续沉默机制
- 缺少早晨事件结算
- 道具系统只是字符串数组，没有栏位、耐久、阵营差异化效果
- 夜晚行动没有按职业顺序执行（狼人→预言家→窃贼→验尸官→结算）
- 没有孤狼冲突规则
- 没有狂狼同归于尽后的平安夜机制
- 没有护身符保护机制

## 目标

实现 doc 中所有已设计机制，除美术资源外，所有功能可用。

## 执行阶段

### Stage 1: 核心数据模型重写

重写 `types.ts`，建立完整的游戏数据模型：

1. **六维属性**: affinity, logic, leadership, deception, stealth, insight (1-10)
2. **阵营**: alignment (lawful/chaotic × good/evil 九宫格)
3. **关系**: trust (-10~+10), friendly (-10~+10) 有向关系
4. **压力值**: stress (-10~+10)
5. **道具系统**: Item 对象（含 durability, ownerTeamEffect, holderTeamEffect）
6. **职业角色**: 完整的职业定义
7. **行动类型**: 所有白天行动、追加行动、夜晚行动
8. **检定系统**: d20 基础

### Stage 2: 模拟器核心重构

重写 `simulator.ts`，修正回合顺序：

1. **正确顺序**: 夜晚 → 早晨事件 → 白天轮流行动 → 连续沉默检测 → 投票 → 放逐结算
2. **夜晚阶段按职业顺序**: 狼人讨论选目标 → 预言家查验 → 窃贼偷取 → 验尸官验尸 → 夜间结算
3. **孤狼冲突**: 普通狼人与孤狼目标相同时杀戮无效
4. **早晨事件**: 公布死亡、预言家公布义务、压力值自然恢复、关系值自然恢复
5. **连续沉默**: 每名玩家回合选择沉默则计数+1，等于存活人数时强制进入投票
6. **追加行动窗口**: 怀疑/袒护后触发追加窗口
7. **投票机制**: 两轮投票、平票处理
8. **道具处理**: 尖牙利爪前提、水晶球碎裂、小偷手套损坏、验尸工具消耗、护身符保护、双刃剑消耗

### Stage 3: 行动系统实现

实现所有行动：

**白天普通行动**:
- 沉默（连续沉默计数）
- 公布身份（含谎言检定）
- 公开信息（公布目标持有道具，含真假检定）
- 观察（洞察 vs 隐蔽，隐蔽性检测）
- 怀疑（洞察/逻辑 vs 隐蔽）
- 袒护（亲和检定）
- 感谢（亲和）
- 号召投票（领导/逻辑）
- 阻止投票（领导/亲和）
- 担保清白（亲和/洞察）
- 强烈指认（洞察/逻辑，触发反驳窗口）
- 全员排除（逻辑/领导，身份冲突时）

**追加行动**:
- 一同怀疑
- 反驳
- 一同袒护

**夜晚行动**:
- kill（狼人，需尖牙利爪）
- check（预言家，需水晶球，狼人碎球）
- steal（窃贼，需手套，一次，偷随机道具）
- inspect（验尸官，需工具，一次，查死者道具）

### Stage 4: 数值与检定系统

实现完整的检定：
- 基础检定: 属性 + d20 vs 难度/目标属性 + d20
- 成功等级: 大失败/失败/成功/大成功
- 含压力修正: 高压力时诡诈/隐蔽减值
- 含阵营修正: 守序/混乱/善良/邪恶对检定和关系的影响
- 含关系修正: 说服时考虑信任值
- 属性影响白天行动，不影响夜晚杀戮

### Stage 5: AI 决策引擎重构

重写 `decision-engine.ts` 和 `belief-system.ts`：
- 使用完整的六维属性做决策
- 使用 d20 检定做决策
- 使用完整的关系值（trust/friendly）做投票和社交决策
- 使用压力值调整行为
- 使用阵营倾向调整行为
- 实现所有职业的特殊决策逻辑

### Stage 6: UI 更新

更新 `GameApp.tsx`：
- 显示六维属性
- 显示阵营
- 显示压力值
- 显示关系网（对选中玩家的信任/友好）
- 显示道具详情（含耐久度）
- 显示更详细的日志（检定结果、关系变化、压力变化）
- 显示当前阶段（早晨/白天/投票/夜晚）
- 显示连续沉默计数

### Stage 7: 集成与验证

确保所有组件能编译通过，游戏逻辑闭环。
