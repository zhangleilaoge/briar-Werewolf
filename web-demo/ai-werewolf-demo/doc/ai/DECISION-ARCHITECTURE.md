# AI 决策架构

## 概述

本文档是 AI 决策的**流程总览**，只讲决策分哪几步、每步做什么。

具体每一步的实现细节（如 SocialContext 的接口定义、mindMultiplier 的公式、温度参数的计算方式）在配套文档 **[MIND-SYSTEM.md](MIND-SYSTEM.md)** 中展开。

一句话概括：狼人杀 AI 采用**策略面具 × 心智驱动乘法评分**——局势先映射到策略面具，候选生成时应用面具适配度，心智 enrich 做乘法加权，Softmax 全分布选择。旧有的策略+插件+意图三系统已整合为统一流程，新增策略面具层作为局势到行动的桥梁。

```
    ┌──────────────────┐    ┌──────────────────┐    ┌──────────────────┐
    │   策略系统        │    │   插件系统        │    │   意图系统        │
    │  (day.ts)        │    │  (evaluateAll)   │    │  (IntentionMgr)  │
    └────────┬─────────┘    └────────┬─────────┘    └────────┬─────────┘

└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │  Layer 3: 人格层（ValueSystem）                                        │
    │  Layer 4: 局势评估层（TimingEvaluator + Crisis + Relation + Social）  │

    │                                                                     │
    │  mindMultiplier =                                                    │
    │    (0.5 + 0.5 × valueAlignment)      -- 价值观匹配（Layer 3）      │
    │    × (0.5 + 0.5 × timingScore)       -- 时机评估（Layer 4）         │
    │    × (0.5 + 0.5 × simulationScore)   -- 心智模拟（Layer 5）         │
    │    × crisisFactor                     -- 身份危机影响（Layer 4）   │
    │    × relationFactor                   -- 关系网络影响（Layer 4）     │
    │    × (0.8 + 0.2 × socialContextBonus) -- 社交情境加成（Layer 4）    │
    │    × capabilityFactor                 -- 能力匹配（Layer 4）         │
    │                                                                     │
    │  totalScore = baseScore × mindMultiplier                           │
    │  baseScore = (score + intentionDrivenBonus + stageWeight + modifiers)│
    │  其中 score = baseScore_raw × maskCompatibility（已含面具缩放）    │
    └─────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  第四步：硬约束过滤（在 mind enrich 之后执行）                              │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │  filterByHardConstraints(enrichedCandidates, context)                 │
    │                                                                     │
    │  - 狼人不得攻击队友 → 拦截 accuse/suspect/call_vote 指向队友        │
    │  - 预言家必须公布查验 → 拦截未跳预言家时的沉默                        │
    │                                                                     │
    │  输出：{ allowed, blocked }                                           │
    │  被拦截的候选也带有 mindData，便于调试和 UI 展示                      │
    └─────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  第五步：去重 + Softmax 全分布选择                                          │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │  1. 排序：按 totalScore 降序                                          │
    │  2. 去重：相同 action+target 只保留最高分                             │
    │  3. Softmax：全分布概率选择，温度参数根据分数差距动态调整              │
    │                                                                     │
    │  温度参数规则：                                                       │
    │  - 分数差距 > 50：温度 = 0.3（高度确定性）                            │
    │  - 分数差距 > 20：温度 = 1.0（正常）                                  │
    │  - 分数差距 > 10：温度 = 2.0（较随机）                                │
    │  - 分数差距 ≤ 10：温度 = 3.0（高度随机）                              │
    └─────────────────────────────────────────────────────────────────────┘


┌─────────────────────────────────────────────────────────────────────────────┐
│  第六步：执行行动                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────────────────────────────────────────────────────────────────┐
    │  选择候选 → 调用对应 Plugin.execute()                               │
    │  → 对抗检定 → 执行效果                                                │
    └─────────────────────────────────────────────────────────────────────┘
```

## 五大系统（一句话说明）

| 系统 | 一句话 | 详细文档 |
|------|--------|----------|
| **面具系统** | 局势→离散策略标签（7面具），决定"什么行动值得考虑" | **[MASK-SYSTEM.md](MASK-SYSTEM.md)** |
| **策略系统** | `day.ts` 已退化为胶水层，只保留阵营判断和兜底逻辑，核心候选生成委托 `CandidateGenerator` | — |
| **插件系统** | 提供具体行动的执行逻辑和对抗检定 | `src/lib/plugins/` |
| **意图系统** | 维护 Desire → Intention → Plan 的 BDI 栈，输出意图匹配加分 | `src/lib/ai/intention/` |
| **心智驱动系统** | 构建 SocialContext + 六维价值观 + 心智模拟 → 对候选做乘法加权 | **[MIND-SYSTEM.md](MIND-SYSTEM.md)** |

面具系统的**七面具定义、适配度表、选择逻辑、层顺序**等详细设计，在 **MASK-SYSTEM.md** 中。

心智驱动系统的**接口定义、评分公式、配置表结构、温度参数计算**等详细设计，在 **MIND-SYSTEM.md** 中。

本文档只保留流程概览。

## 附录：分数计算示例

假设一个候选（attack 面具下的 ACCUSE）：
- baseScore_raw = 400（原始基础分）
- maskCompatibility = 0.80（attack 对 ACCUSE 的适配度）
- score = 400 × 0.80 = 320（面具缩放后的基础分）
- intentionDrivenBonus = 200（主意图匹配）
- stageWeight = 100（社交阶段）
- modifiers.total = 0（无修正）
- valueAlignment = 0.6, timingScore = 0.5, simulationScore = 0.5
- crisisFactor = 1.0, relationFactor = 1.0, socialContextBonus = 1.0, capabilityFactor = 1.0

计算过程：
```
baseScore = 320 + 200 + 100 + 0 = 620

mindMultiplier =
  (0.5 + 0.5 × 0.6) ×     -- 0.80
  (0.5 + 0.5 × 0.5) ×     -- 0.75
  (0.5 + 0.5 × 0.5) ×     -- 0.75
  1.0 × 1.0 × (0.8 + 0.2 × 1.0) × 1.0  -- 1.0
  = 0.80 × 0.75 × 0.75 × 1.0 = 0.45

totalScore = 620 × 0.45 = 279 → 279
```

UI 展示：`总分：279 = 620(基础分) × 0.45(心智因子)`，其中基础分包含 `320(score) + 200(意图) + 100(阶段)`，score 已含 attack 面具的 0.80 适配度。
