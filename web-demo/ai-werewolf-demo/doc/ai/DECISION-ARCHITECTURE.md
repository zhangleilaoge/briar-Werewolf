# AI 决策架构

## 概述

狼人杀 AI 使用**三系统并行**的决策架构：策略系统、插件系统、意图系统。三者产生的候选行动在同一评分体系中竞争，最终通过加权随机选择。

## 三大系统详解

### 1. 策略系统 (Strategy System)

**文件**: `src/lib/ai/strategies/`

**职责**: 根据游戏状态和规则生成候选行动

**阶段权重**:
| 阶段 | 权重 | 含义 |
|------|------|------|
| duty | 1000 | 职业义务（预言家必须公布查验） |
| survival | 800 | 生存（被攻击时自保） |
| information | 500 | 信息（观察、查验） |
| social | 100 | 社交（跟随号召、袒护） |

**核心策略**:
- `VillagerDayStrategy`: 村民白天行为
- `WerewolfCamouflageStrategy`: 狼人伪装行为
- `ProphetClaimStrategy`: 预言家公布义务
- `VoteStrategy`: 投票决策

---

### 2. 插件系统 (Plugin System)

**文件**: `src/lib/plugins/`

**职责**: 提供具体行动的执行逻辑和 AI 评估

**插件类型**:
| 类型 | 目录 | 说明 |
|------|------|------|
| item | `items/` | 装备道具（水晶球、利爪等） |
| action | `actions/` | 行为动作（怀疑、袒护、号召投票等） |
| trait | `traits/` | 特质（孤狼等） |

**核心接口**:
```typescript
interface ActionProvider {
  id: string;
  type: 'item' | 'action' | 'trait';
  getAvailableActions(player, context): ActionDefinition[];
  execute(params): ActionResult;
  evaluate?(context): DecisionCandidate[];  // AI 评估
}
```

---

### 3. 意图系统 (Intention System)

**文件**: `src/lib/ai/intention/`

**职责**: 维护长期目标，生成符合目标的候选行动

**意图类型**:
| 类型 | 说明 | 示例计划 |
|------|------|----------|
|潜伏 | 狼人隐藏身份 | 沉默→怀疑好人→投票 |
| 攻击 | 消灭对手 | 怀疑→号召投票→投票 |
| 保护 | 保护同伴 | 袒护→担保→投票 |
| 揭示 | 预言家公布查验 | 公布身份→号召投票 |

**评分机制**:
- **基础分**: 意图的 priority 值（由 desire-engine 计算）
- **意图匹配**: 主意图 +200，次意图 +50
- **阶段权重**: 统一 +100（social 阶段）

---

## 评分公式详解

```
总分 = 基础分 + 意图匹配 + 阶段权重 + 阵营修正 + 压力修正 + 关系修正
```

| 组成部分 | 来源 | 说明 |
|----------|------|------|
| 基础分 | 策略/插件/意图 | 原始分数 |
| 意图匹配 | 意图系统 | 匹配主意图 +200，次意图 +50 |
| 阶段权重 | 策略引擎 | 根据阶段类型加分（见上表） |
| 阵营修正 | 行为修正常量 | 阵营倾向加成/减益 |
| 压力修正 | 行为修正常量 | 压力状态加成/减益 |
| 关系修正 | 行为修正常量 | 社交关系加成/减益 |
